深入理解 Objective-C Runtime

Objective-C 是一门动态语言，其动态特性的核心就是 Runtime。理解 Runtime 不仅有助于深入理解 Objective-C，还能让我们编写出更灵活的代码。本文将深入探讨 Runtime 的核心概念和实际应用。

Runtime 概述

Objective-C Runtime 是一套 C 语言的 API，负责：

  • 消息发送与转发
  • 类和对象的创建
  • 方法的动态解析
  • 类型编码

消息发送流程

  1. 通过 isa 找到类对象
  2. 在类的方法缓存（cache）中查找
  3. 如果没找到，在方法列表（methodLists）中查找
  4. 如果没找到，沿着继承链向上查找
  5. 如果还是没找到，进入消息转发流程

消息转发

当对象无法响应消息时，Runtime 提供了三次拯救机会：

第二步：快速转发

第三步：完整转发

总结

Runtime 是 Objective-C 的核心，掌握它可以：

  1. 深入理解消息发送机制
  2. 实现 AOP（面向切面编程）
  3. 为系统类添加功能
  4. 实现动态化能力

但也要注意：

  • Method Swizzling 要在 +load 中执行
  • 使用 dispatch_once 保证只执行一次
  • 过度使用会降低代码可读性

建议在必要时才使用 Runtime 黑魔法，保持代码的可维护性。
