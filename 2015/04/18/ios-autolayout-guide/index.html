Auto Layout 实战指南：告别 Frame 布局

随着 iPhone 6 和 iPhone 6 Plus 的发布，iOS 设备的屏幕尺寸越来越多样化。传统的 Frame 布局已经难以适应多屏幕需求，Auto Layout 成为了 iOS 开发的必备技能。

为什么使用 Auto Layout

在 Auto Layout 之前，我们使用 Frame 布局：

这种方式的问题：

  • 不同屏幕尺寸需要手动适配
  • 横竖屏切换需要重新计算
  • 代码难以维护

Auto Layout 通过约束（Constraints）描述视图之间的关系，系统自动计算最终位置和大小。

基本概念

约束（Constraint）

约束描述视图的位置和大小关系，包含以下元素：

例如：view.leading = superview.leading + 20

常用属性

  • 位置：leading、trailing、top、bottom、centerX、centerY
  • 大小：width、height

代码创建约束

方式一：NSLayoutConstraint

方式二：VFL（Visual Format Language）

VFL 使用字符串描述约束，更加直观：

VFL 语法说明

符号 含义
` `
- 标准间距（8pt）
-20- 指定间距
[view] 视图
[view(100)] 视图宽/高 100
[view(>=100)] 宽/高大于等于 100

约束优先级

约束有优先级（1-1000），默认 1000（必须满足）：

常用优先级：

  • UILayoutPriorityRequired = 1000
  • UILayoutPriorityDefaultHigh = 750
  • UILayoutPriorityDefaultLow = 250

Intrinsic Content Size

某些控件有固有大小（Intrinsic Content Size），如 UILabel、UIButton：

Content Hugging & Compression Resistance

  • Content Hugging：抗拉伸优先级，值越高越不容易被拉伸
  • Compression Resistance：抗压缩优先级，值越高越不容易被压缩

总结

Auto Layout 的核心要点：

  1. 设置 translatesAutoresizingMaskIntoConstraints = NO
  2. 每个视图需要足够的约束确定位置和大小
  3. 利用 Intrinsic Content Size 减少约束
  4. 善用优先级解决冲突
  5. 使用 VFL 提高代码可读性
