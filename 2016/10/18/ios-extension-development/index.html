iOS 扩展开发指南：Today Widget 与 Share Extension

iOS 8 引入了 App Extension，允许应用在系统的各个位置提供功能。本文将介绍两个最常用的扩展：Today Widget（通知中心小组件）和 Share Extension（分享扩展）。

App Extension 概述

App Extension 是独立的二进制文件，运行在独立的进程中，但与主应用共享签名和部分资源。常见的扩展类型包括：

  • Today Widget：通知中心小组件
  • Share Extension：分享扩展
  • Action Extension：操作扩展
  • Photo Editing Extension：照片编辑扩展
  • Custom Keyboard：自定义键盘
  • 等等…

扩展开发注意事项

1. 内存限制

扩展的内存限制比主应用小得多，需要注意：

  • Today Widget：约 16MB
  • Share Extension：约 120MB

2. 不可用的 API

某些 API 在扩展中不可用：

  • UIApplication.shared（但可以用 extensionContext?.open(url:)）
  • HealthKit（部分）
  • 后台任务
  • AirDrop

3. 共享代码

可以创建一个共享的 Framework 来复用代码：

  1. 创建 Cocoa Touch Framework Target
  2. 将共享代码放入 Framework
  3. 在主应用和扩展中都链接这个 Framework

总结

App Extension 大大扩展了 iOS 应用的能力边界：

  1. Today Widget 让用户快速查看信息
  2. Share Extension 让用户方便地分享内容到你的应用

开发扩展时需要注意：

  • 配置 App Groups 实现数据共享
  • 注意内存和 API 限制
  • 使用 Framework 共享代码

随着 iOS 系统的发展，扩展的能力越来越强，建议根据应用特点选择合适的扩展类型。
